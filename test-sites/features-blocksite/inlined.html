BlockSite features

Cool features

  • Browse any blocksite, no wallet asked.
  • Completely hosted on chain, will be availble forever to everyone‚Ñ¢Ô∏è (or until the owner decides to destroy the object).
  • One-click deploy of (simple) blocksites.
  • Very cheap thanks to Sui.
  • Link across blockpages, and seamlessly with Web2 websites.

Current problems

  • Maximum object size limits the size of each blocksite (~250KB compressed).
  • The blocksite is an opaque compressed blob: Updating is almost as costly as publishing an entirely new blocksite.
  • The blocksite portal is just a PoC.
  • Cannot connect to the sui wallet!!

Ideas for future improvement

  • Serve js/css from other blocks as plaintext files.
  • Linking is quite cumbersome at the moment (need to create the Objects to get the IDs, and then update them with the content). Create tooling for that (Note: Started).
  • Implmement the same functionality of the "blocksite portal" in a bookmarklet, so that hosting the portal is not required anymore.
  • Add the possiblity to store the blocksite in multiple objects to get around the maximum object size limit
  • Create tooling to split a single logical "webpage" into multiple Sui objects, according to some template. This could be interesting, e.g., to allow "multi-owner websites". For instance, a news site where individual articles are actually owned by individual authors, which can load anything in their "section" (the Sui object they own that is then linked in the blocksite).
